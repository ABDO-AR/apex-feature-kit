## Overview

Create a comprehensive `README.md` for the `@abdoar/apex-feature-kit` project — a global Node.js CLI tool for Feature-Driven Development (FDD) that scaffolds the `.features/` directory, injects AI slash commands into IDE platforms (Kilo Code & Cursor), and auto-syncs completion status. The README serves as the project's landing page on GitHub and npm, so it must clearly explain what the tool does, how to install it, how to use it, and how the FDD workflow operates. A developer reading this README should be able to go from zero to running `apex-feature-kit init all` in under 2 minutes.

**Branding alignment with abdoar.com:** The README must reflect the Abdo AR brand identity. Abdo AR (Abdul Rahman Fathi El-Sayed) is a Co-Founder & CTO, digital entrepreneur, and software engineer whose mission is helping startups build market-ready digital products — fast, scalable, and cost-effective. The README should carry this same startup-empowering, professional-yet-approachable tone. The tool itself embodies this mission: it accelerates feature delivery by giving AI agents a structured blueprint, turning ideas into production-ready features with minimal overhead.

## Technical Context

- **Tech Stack**: Node.js (ES2022), TypeScript 6, Commander.js for CLI, Zod 4 for validation, fs-extra for filesystem, js-yaml for YAML, chalk for terminal styling
- **Project Structure**: CLI with only 2 commands (`init`, `sync`). Source in `src/` with `commands/`, `core/`, `templates/`, and `ui/` directories. Feature specs are managed entirely by AI agents through slash commands — the CLI handles scaffolding and syncing only.
- **Existing Patterns**: The project uses ES modules with `NodeNext` module resolution. Imports use `.js` extensions. The `.features/` directory contains `tree.yaml`, `instructions.md`, and `specs/`. Slash commands live in `.kilo/commands/` or `.cursor/commands/`.
- **Related Features**: None — this is the first feature spec.
